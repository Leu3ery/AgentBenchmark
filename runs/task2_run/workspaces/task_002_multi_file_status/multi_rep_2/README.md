# Multi-file Status Fixture

This task simulates a small but realistic cross-layer status update.

Useful paths:
- `issue.txt` contains the visible product request
- `backend/statuses.py` contains the backend catalog and helper functions
- `validation/schema.py` contains payload validation
- `api/serializer.py` formats status values for API output
- `tests/test_status.py` contains public regression tests
