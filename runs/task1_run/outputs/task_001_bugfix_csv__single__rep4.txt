Summary of what I changed and why
- Bug: parse_csv_line split on every comma, even commas inside quoted fields. That caused fields like '"beta,gamma"' to be split into two fields.
- Fix: only treat commas as separators when not inside an open quoted field (i.e., when in_quotes is False).
- Files changed:
  - repo/parser.py — fixed the comma-splitting logic so commas inside quotes are preserved.
  - repo/__init__.py — added a small package init so repo can be imported by tests.
  - tests/test_parser.py — made tests runnable in the workspace by adding the project root to sys.path (this is a small test harness adjustment so pytest can import repo).
