Bug summary
- The parser previously split on every comma regardless of whether it was inside a quoted field. This caused quoted fields containing commas (e.g. "beta,gamma") to be split into multiple fields.

Code change summary (targeted)
- repo/parser.py
  - parse_csv_line now tracks an in_quotes flag and:
    - Toggles in_quotes when a double-quote (") is encountered and skips the quote character.
    - Treats a comma as a separator only when not in_quotes; when in_quotes the comma is appended to the current field.
  - This change keeps commas inside quoted fields as part of the field instead of splitting.

Remaining risks / limitations
- Escaped quotes inside quoted fields are not handled (CSV uses "" to represent a literal " inside a quoted field). The current code toggles in_quotes on every quote, so double double-quotes will not be parsed correctly.
- Multiline quoted fields are not supported.
- This is a small, targeted parser; it does not attempt to implement the full CSV RFC. If full CSV compliance is required, consider using Python's built-in csv module or extend the parser to handle escaped quotes and newlines inside quoted fields.
