# Fake Email Workflow Fixture

This task simulates a small inbox triage and reply workflow.

Useful files:
- `context.md` explains the task context
- `policy.md` defines when a reply is needed
- `faq.md` contains the approved answers you can reuse
- `emails/inbox.json` contains the fake inbox
- `output_template.md` suggests how to summarize your actions

Important:
- replies must be saved with the fake reply tool
- do not reply to non-actionable confirmation or thank-you messages
