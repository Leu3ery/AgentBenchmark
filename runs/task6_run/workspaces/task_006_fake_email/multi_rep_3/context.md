# Inbox Workflow Context

This task simulates a lightweight customer-support inbox workflow.

Available context:
- `policy.md` explains when to reply and how to phrase responses
- `faq.md` contains the approved answers you can use
- `emails/inbox.json` contains the messages to triage

Expected behavior:
- reply to actionable questions
- skip messages that do not need follow-up
- save all reply drafts through the fake reply tool
